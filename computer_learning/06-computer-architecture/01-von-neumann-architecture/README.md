# 01 - 冯·诺依曼架构（Von Neumann Architecture）

> ✅ 学习状态：进行中

## 1.1 什么是冯·诺依曼架构？

### 历史背景

1945 年，数学家 **冯·诺依曼（John von Neumann）** 在 EDVAC 报告中提出了**存储程序（Stored Program）**概念，奠定了现代计算机的基本架构。

在此之前，计算机（如 ENIAC）的程序是通过插拔线缆和开关来"编程"的——改程序等于改硬件。冯·诺依曼的核心洞见是：**程序也是一种数据，可以和数据一起存在内存里**。

### 五大组件

冯·诺依曼架构将计算机分为五个基本部分：

```
┌─────────────────────────────────────────────────────┐
│                    系统总线（Bus）                     │
│         数据总线 │ 地址总线 │ 控制总线                  │
└──────┬──────────┴──────────┴──────────┬─────────────┘
       │                                │
  ┌────▼────┐                      ┌───▼────┐
  │  输入设备 │                      │ 输出设备  │
  │ 键盘/鼠标 │                      │ 显示器    │
  │ 网卡/磁盘 │                      │ 打印机    │
  └────┬────┘                      └───▲────┘
       │                                │
       │    ┌──────────────────┐        │
       └────►      内存(Memory)  ├────────┘
            │  存储程序 + 数据    │
            └──────┬───────────┘
                   │
       ┌───────────▼───────────┐
       │         CPU            │
       │  ┌──────┐ ┌─────────┐ │
       │  │控制单元│ │ ALU     │ │
       │  │(CU)   │ │(运算器)  │ │
       │  └──────┘ └─────────┘ │
       │  ┌──────────────┐     │
       │  │   寄存器组     │     │
       │  │  PC|IR|ACC|.. │     │
       │  └──────────────┘     │
       └───────────────────────┘
```

| 组件 | 英文 | 作用 | 类比 |
|------|------|------|------|
| **输入设备** | Input | 把数据和程序送入计算机 | 眼睛、耳朵 |
| **输出设备** | Output | 把计算结果送出计算机 | 嘴巴、手 |
| **存储器** | Memory | 存放程序和数据 | 草稿纸 |
| **控制器** | Control Unit | 指挥协调各部件工作 | 大脑皮层 |
| **运算器** | ALU | 算术运算 + 逻辑运算 | 大脑计算中枢 |

## 1.2 存储程序（Stored Program）概念

### 核心思想

**指令和数据以相同的二进制形式存放在同一存储器中**。这意味着：
- 程序可以像数据一样被读取、修改
- 程序可以生成程序（编译器、JIT 的本质）
- 无需改动硬件就能切换任务——换个程序就行

```
内存中的存储：
地址    内容             含义
0x0000: 01001001...    指令 1（LOAD R1, [A]）
0x0004: 00101001...    指令 2（ADD R1, R2）
0x0008: 01010000...    指令 3（STORE [B], R1）
0x000C: 00000000...    （下一条指令...）
...
0x1000: 0000002A       数据 A = 42
0x1004: 00000010       数据 B = 16
```

> 🔑 指令和数据混在一起，CPU 靠 **PC（程序计数器）** 来区分"当前执行到哪了"。

### 指令周期

每条指令的执行都遵循相同的节奏：

```
取指（Fetch）  →  译码（Decode）  →  执行（Execute）  →  写回（Writeback）
     ↑                                                      │
     └──────────────────────────────────────────────────────┘
                     PC + 1，取下一条
```

## 1.3 系统总线（System Bus）

五大组件之间通过**总线（Bus）**通信。总线就是一组并行的导线。

```
┌────────────────────────────────────────────┐
│              数据总线（Data Bus）              │
│  双向传输数据，宽度决定"一次能传多少"            │
│  常见：8位 / 16位 / 32位 / 64位               │
├────────────────────────────────────────────┤
│             地址总线（Address Bus）            │
│  CPU → 内存，指定要访问的内存地址               │
│  n 条地址线 → 可寻址 2^n 个内存单元            │
│  32位地址总线 → 最大 4GB 内存（2^32）           │
├────────────────────────────────────────────┤
│             控制总线（Control Bus）            │
│  传输控制信号：读/写、中断请求、时钟信号等        │
│  由控制单元（CU）管理                          │
└────────────────────────────────────────────┘
```

## 1.4 冯·诺依曼 vs 哈佛架构

| | 冯·诺依曼 | 哈佛架构 |
|---|-----------|----------|
| **指令和数据** | 共用一条总线、同一内存 | 分开的总线和存储器 |
| **优点** | 简单、灵活、硬件成本低 | 可以同时取指和取数（更快） |
| **缺点** | **冯·诺依曼瓶颈**（总线竞争） | 硬件复杂、成本高 |
| **应用** | PC、服务器、通用计算机 | DSP（数字信号处理器）、单片机、ARM Cortex-M |

```
冯·诺依曼：                        哈佛架构：
┌──────┐                          ┌──────────┐
│ CPU  │                          │   CPU    │
└──┬───┘                          └┬───┬────┘
   │ 共享总线                        │   │ 分开的总线
   │                     ┌─────────┘   └────────┐
┌──▼──────────┐       ┌──▼────────┐  ┌─────────▼──┐
│  统一内存     │       │ 指令存储器  │  │ 数据存储器   │
│ (指令 + 数据) │       │ (只存储指令) │  │ (只存储数据) │
└─────────────┘       └───────────┘  └────────────┘
```

> 现代 CPU 内部往往混合使用：对外是冯·诺依曼，片内 L1 Cache 分离指令和数据（哈佛方式），叫**改进的哈佛架构**。

---

## 1.5 冯·诺依曼瓶颈

由于指令和数据共享同一条总线通往内存，CPU 的速度受限于内存访问速度。这就是著名的**冯·诺依曼瓶颈**。

```
CPU 速度增长（摩尔定律） vs  内存速度增长（相对缓慢）

速度
 ↑                              ╱ CPU
 │                             ╱
 │                           ╱  ← 差距越来越大
 │                         ╱
 │                       ╱
 │          ════════════  ← 内存（增长缓慢）
 │
 └──────────────────────────────→ 时间
```

**解决思路**：
1. **缓存（Cache）** — 把常用数据放在离 CPU 更近的高速存储器中
2. **预取（Prefetching）** — CPU 提前把可能用到的指令/数据取到缓存
3. **多核/并行** — 多条路径同时访问内存

> 这些会在 [04 - 存储层次](../04-memory-hierarchy/) 和 [05 - 流水线与并行](../05-pipeline-and-parallelism/) 中详细展开。

## 📝 本章速查

```
冯·诺依曼架构 = 存储程序概念 + 五大组件

五大组件：输入 → 内存 ←→ CPU(控制器 + ALU) → 输出

存储程序：指令和数据存放在同一内存，以二进制形式存在

指令周期：取指 → 译码 → 执行 → 写回

三大总线：数据总线（传数据）、地址总线（传地址）、控制总线（传信号）

哈佛架构：指令和数据分开存储 → 更快，但更复杂

冯·诺依曼瓶颈：CPU 和内存速度不匹配 → 靠 Cache 缓解
```
