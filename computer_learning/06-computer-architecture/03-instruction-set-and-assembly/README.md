## 3.1 什么是 ISA（指令集架构）？

### 计算机的"语言层级"

```
┌────────────────────────────────────────┐
│  高级语言: Python, C++, Rust...         │  ← 人类友好
├────────────────────────────────────────┤
│  汇编语言: MOV, ADD, JMP...            │  ← 符号化指令
├────────────────────────────────────────┤
│  机器码: 01001010 11000011...          │  ← CPU 真正执行的
├────────────────────────────────────────┤
│  微指令(微码): 门级控制信号              │  ← CPU 内部实现
└────────────────────────────────────────┘
```

**ISA（Instruction Set Architecture）** 是软件和硬件之间的**契约**：
- 它定义了 CPU 支持哪些指令、指令的格式、寄存器、寻址方式
- 软件只知道 ISA，不关心 CPU 内部怎么实现
- 同一个 ISA 可以有不同的微架构实现（Intel Core i9 和 i3 都是 x86 ISA，但内部实现完全不同）

## 3.2 CISC vs RISC

这是指令集设计的两大流派。

```
                    CISC                              RISC
              ┌──────────────┐                 ┌──────────────┐
              │  复杂指令     │                 │  简单指令     │
              │  计算机       │                 │  计算机       │
              │              │                 │              │
              │ 一条指令做    │                 │ 一条指令做    │
              │ 很多事情      │     ←──VS──→    │ 一件小事      │
              │              │                 │              │
              │ 指令不等长    │                 │ 指令定长      │
              │ 内存-内存操作  │                 │ 只有LOAD/STORE│
              └──────────────┘                 └──────────────┘
                    x86                           ARM / RISC-V
```

| 特征 | CISC (x86) | RISC (ARM/RISC-V) |
|------|------------|-------------------|
| **指令条数** | 很多（500+） | 少（几十~一百多） |
| **指令长度** | 可变（1~15 字节） | 固定（2 或 4 字节） |
| **寻址方式** | 多种 | 少（通常 3~5 种） |
| **访存方式** | 任何指令都可能访问内存 | 只有 LOAD/STORE 访存 |
| **硬件复杂度** | 复杂 | 相对简单 |
| **编译器友好度** | 较难优化 | 更好优化 |
| **功耗** | 较高 | 较低（适合移动设备） |

### 一个直观对比

计算 `C = A + B`，假设 A、B、C 都在内存中：

```
RISC (ARM 风格):                    CISC (x86 风格):
LOAD  R0, [A]        3 条简单指令   MOV  AX, [A]      1 条指令完成！
LOAD  R1, [B]       (每条约 4 字节)   ADD  AX, [B]      (但指令更长)
ADD   R2, R0, R1                      MOV  [C], AX
STORE [C], R2

12 字节                              ~10 字节（不等长）
```

> 💡 实际上现代 x86 CPU 内部把复杂 CISC 指令**拆成类 RISC 微操作（μops）**来执行。所以 CISC 对外复杂，但内部实现也借鉴了 RISC 思想。

## 3.3 指令的解剖结构

一条机器指令通常包含：

```
┌─────────────┬──────────┬──────────┬──────────┬──────────┐
│   操作码     │  操作数1  │  操作数2  │  操作数3  │ 立即数/偏移 │
│  (Opcode)   │ (Operand) │          │          │          │
└─────────────┴──────────┴──────────┴──────────┴──────────┘
     做什么          对谁做                          附加信息
```

### 以 ARM 为例

```
ADD R0, R1, R2    →  操作码=ADD  操作数=R0, R1, R2
    ↓                  ↓
   | 31-28 | 27-26 | 25-24 | 23-20 | 19-16 | 15-12 | 11-0 |
   | cond  |  00   |  I    | opcode|  Rn   |  Rd   | operand2 |
    ↓      ↓       ↓       ↓       ↓       ↓       ↓
   条件码  ALU类  是否立即数 具体运算  源寄存器 目标寄存器 源操作数2
```

## 3.4 指令的三大类

### 数据处理指令

操作寄存器中的值，结果放到寄存器。

```asm
ADD  R0, R1, R2   ; R0 = R1 + R2
SUB  R3, R3, #1   ; R3 = R3 - 1  ( # 表示立即数)
MUL  R4, R5, R6   ; R4 = R5 × R6
AND  R7, R7, #0xFF ; R7 = R7 & 0xFF (位掩码)
ORR  R8, R8, #0x01 ; R8 = R8 | 0x01 (设某位为1)
LSL  R9, R9, #2   ; R9 = R9 << 2  (左移 = 乘以4)
```

### 访存指令

在寄存器和内存之间搬移数据。

```asm
LDR  R0, [R1]      ; R0 = 内存[R1]          (间接寻址)
LDR  R0, [R1, #4]  ; R0 = 内存[R1 + 4]      (基址+偏移)
STR  R2, [R1]      ; 内存[R1] = R2          (存一个寄存器)
STR  R2, [R1, #8]! ; 内存[R1+8] = R2, R1+=8 (前索引写回)
LDMIA R0!, {R1-R4} ; 多寄存器加载（批量读）
```

### 控制流指令

改变程序的执行路径。

```asm
B    label          ; 无条件跳转到 label
BEQ  label          ; 如果 Z=1 (相等)，跳转
BNE  label          ; 如果 Z=0 (不等)，跳转
BLT  label          ; 如果 N≠V (小于)，跳转
BGT  label          ; 如果 Z=0 且 N=V (大于)，跳转
BL   func           ; 函数调用 (LR=返回地址，跳转到 func)
BX   LR             ; 函数返回 (跳转到 LR 中的地址)
```

## 3.5 寻址模式（Addressing Modes）

所谓"寻址"，就是指令如何找到它的操作数。

| 模式 | 示例 | 操作数来源 | 用途 |
|------|------|------------|------|
| **立即数** | `ADD R0, #42` | 指令中直接给出值 | 常量 |
| **寄存器直接** | `ADD R0, R1` | 寄存器的值 | 最常用 |
| **寄存器间接** | `LDR R0, [R1]` | 寄存器→地址→内存 | 指针解引用 |
| **基址+偏移** | `LDR R0, [R1, #8]` | R1+8→地址→内存 | 结构体字段、数组 |
| **基址+变址** | `LDR R0, [R1, R2]` | R1+R2→地址→内存 | 数组索引 |
| **PC 相对** | `B label` | PC + offset | 条件/无条件跳转 |
| **自动递增** | `LDR R0, [R1, #4]!` | 先计算 R1+4，再 R1=R1+4 | 栈操作 |
| **栈寻址** | `PUSH {R0}` | SP 隐式指向 | 函数调用 |

## 3.6 调用约定（Calling Convention）

当函数 A 调用函数 B 时，参数怎么传？返回值怎么拿？谁来保存寄存器？这些规则就是调用约定。

### ARM AAPCS（ARM 架构过程调用标准）

```
寄存器分工：
  R0-R3   : 函数参数 1~4（多余的放栈上）
  R0-R1   : 函数返回值
  R4-R11  : 被调用者保存（callee-saved）—— 子函数用之前必须先保存
  R13 (SP): 栈指针
  R14 (LR): 链接寄存器（存返回地址）
  R15 (PC): 程序计数器

函数调用过程：
  调用方:                 被调用方:
  BL  func                PUSH {R4, LR}     ; 保存旧值
  ...                     ...函数体...
                          POP  {R4, PC}     ; 恢复并返回
```

## 3.7 汇编编程实例

```asm
;; 计算 1+2+...+N 的汇编程序（ARM 风格伪代码）

        MOV  R0, #0       ; sum = 0
        MOV  R1, #10      ; N = 10

loop:   CMP  R1, #0       ; counter == 0?
        BEQ  done         ; 如果是，结束
        ADD  R0, R0, R1   ; sum += counter
        SUB  R1, R1, #1   ; counter--
        B    loop         ; 继续循环

done:   STR  R0, [result] ; 存结果到内存
        BX   LR           ; 返回
```

## 3.8 汇编与高级语言的对应

| 高级语言 | 汇编对应 |
|----------|----------|
| `int x = 5;` | `MOV R0, #5` |
| `x = y + z;` | `ADD R0, R1, R2` |
| `x = *ptr;` | `LDR R0, [R1]` |
| `*ptr = x;` | `STR R0, [R1]` |
| `if (x == y)` | `CMP R0, R1` → `BEQ` |
| `for (i=0; i<n; i++)` | `CMP` + `BGE` + `B` |
| `return x;` | `MOV R0, R1` → `BX LR` |
| `x = y << 2;` | `LSL R0, R1, #2` |

```
ISA = 硬件和软件之间的契约（指令集 + 寄存器 + 寻址模式 + ...）

CISC (x86): 复杂指令，不等长，功能强大但功耗高

```
