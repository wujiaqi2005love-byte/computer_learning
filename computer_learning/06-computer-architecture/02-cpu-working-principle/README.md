## 2.1 CPU 内部结构

### CPU 三大核心部件

```

│                    CPU                        │
│                                               │
│  ┌─────────────────┐  ┌───────────────────┐  │
│  │   控制器 (CU)     │  │   运算器 (ALU)     │  │
```

## 2.2 寄存器的角色

寄存器是 CPU 内部最快的存储单元，纳秒级访问。

### 关键寄存器一览

| 寄存器 | 全称 | 作用 | 类比 |
|--------|------|------|------|
| **PC** | Program Counter | 指向下一条要取出的指令地址 | 书签——读到哪了 |
| **IR** | Instruction Register | 存放当前正在执行的指令 | 当前页的内容 |
| **MAR** | Memory Address Register | 存放即将访问的内存地址 | 你要去哪个书架 |
| **MBR/MDR** | Memory Buffer/Data Register | 存放从内存读出/即将写入的数据 | 你从书架拿/放的书 |
| **ACC** | Accumulator | 存放 ALU 运算结果 | 计算器的显示屏 |
| **SP** | Stack Pointer | 指向栈顶 | — |
| **FLAGS/PSR** | Program Status Register | 状态标志（零、进位、溢出等） | — |

### 为什么区分 MAR 和 MBR？

它们是 CPU 和内存之间通信的桥梁：

```
CPU 要读内存地址 0x1000 的数据：

1. CPU → MAR = 0x1000    （"我要读 0x1000"）
2. CU 发 READ 信号给控制总线
3. MAR 值放到地址总线上
4. 内存响应，把 0x1000 的数据放到数据总线上
5. 数据进入 MBR           （"数据拿到了"）
6. MBR → ACC 或其他寄存器
```

## 2.3 指令周期：取指-译码-执行

每条指令的执行都严格遵循这个循环：

```
         ┌──────────────────────────────┐
         │                              │
         ▼                              │
    ┌─────────┐    ┌─────────┐    ┌─────────┐    ┌──────────┐
    │  Fetch  │ →  │ Decode  │ →  │ Execute │ →  │ Writeback│
    │  取指    │    │  译码    │    │  执行    │    │  写回     │
    └─────────┘    └─────────┘    └─────────┘    └────┬─────┘
         ▲                                            │
         │              PC = PC + 1                    │
         └────────────────────────────────────────────┘
```

### Fetch（取指）细节

```
1. MAR ← PC           // 把 PC 里的地址送到地址总线
2. MBR ← Memory[MAR]  // 从内存读指令（通过数据总线）
3. IR  ← MBR          // 指令移到指令寄存器
4. PC  ← PC + 1       // PC 指向下一条（或 +4，取决于指令长度）
```

### Decode（译码）

控制单元（CU）解析 IR 中的指令：
- **操作码（Opcode）**：告诉 CU "做什么"（加、减、读、写、跳转...）
- **操作数（Operand）**：告诉 CU "对谁做"（寄存器编号、内存地址、立即数...）

```
指令示例: ADD R1, R2, R3   →  操作码 = ADD，操作数 = R1, R2, R3
                             含义：R1 = R2 + R3
```

### Execute（执行）

根据译码结果，ALU 执行实际操作。不同指令的执行路径不同：

| 指令类型 | 执行动作 |
|----------|----------|
| **算术/逻辑** | ALU 计算，结果存 ACC 或目标寄存器 |
| **访存（LOAD）** | MAR←地址，读内存，数据→目标寄存器 |
| **访存（STORE）** | MAR←地址，寄存器数据→MBR，写内存 |
| **分支/跳转** | 如果条件满足，修改 PC 为目标地址 |
| **I/O** | 通过 I/O 端口与外部设备通信 |

## 2.4 一个完整指令的执行动画

以 `ADD R1, [0x1000]`（把 R1 加上内存地址 0x1000 处的值）为例：

```
阶段      PC      IR        MAR      MBR      R1      ACC
──────────────────────────────────────────────────────────
初始      0x0000  ?         ?        ?        10      ?
FETCH     0x0004  ADD ...   0x0000   ADD ...  10      ?
DECODE    0x0004  ADD ...   0x0000   ADD ...  10      ?
EXEC:读地址 0x0004  ADD ...  0x1000   (内存值)  10      ?
EXEC:加法 0x0004  ADD ...   0x1000   (内存值)  ?      10+(内存值)
WRITEBACK 0x0004  ADD ...   0x1000   (内存值)  10+(内存值) 10+(内存值)
```

## 2.5 时钟与频率

### 时钟信号

CPU 内部有一个**时钟发生器**（通常是石英晶体振荡器），产生方波信号：

```
时钟信号:
    ┌─┐   ┌─┐   ┌─┐   ┌─┐   ┌─┐
────┘ └───┘ └───┘ └───┘ └───┘ └───
    ←── 1 个时钟周期 ──→
```

| 术语 | 含义 | 关系 |
|------|------|------|
| **时钟频率** | 每秒振荡次数 | 3 GHz = 30 亿次/秒 |
| **时钟周期** | 一次振荡的时间 | 1 / 频率，3GHz → 0.33 ns |
| **CPI** | 每条指令平均周期数 | 不同指令 CPI 不同 |
| **IPC** | 每个周期执行指令数 | CPI 的倒数 |

> ⚠️ 误区澄清：CPU 主频高 ≠ 性能一定好。还要看**每周期能做多少事（IPC）**。一个 2GHz 但每个周期执行 4 条指令的 CPU，比一个 4GHz 但每个周期只执行 1 条的更强。

## 2.6 ALU 深入

### 算术逻辑单元的结构

```
        ┌──────────────────────┐
 输入端 A ──┤                      ├── 输出结果
 输入端 B ──┤       ALU            ├── 状态标志
            │                      │   (Zero, Carry,
 功能选择 ──┤  ┌────┬────┬────┬──┐ │    Overflow, Negative)
 (3-bit)    │  │加法│减法│AND │OR │ │
            │  │器  │器  │门  │门 │ │
            │  └────┴────┴────┴──┘ │
            └──────────────────────┘
```

### ALU 能做什么？

| 类别 | 操作 | 说明 |
|------|------|------|
| **算术** | ADD, SUB, MUL, DIV | 加减乘除 |
| **逻辑** | AND, OR, NOT, XOR | 按位逻辑 |
| **移位** | SHL, SHR, ROL, ROR | 左移、右移、循环移位 |
| **比较** | CMP | 比较两数，只设标志不存结果 |

### 状态标志（FLAGS）

每次 ALU 运算后，会设置相应的标志位：

| 标志 | 含义 | 触发条件 |
|------|------|----------|
| **Z (Zero)** | 零标志 | 结果为 0 |
| **C (Carry)** | 进位/借位标志 | 加法溢出、减法借位 |
| **N (Negative)** | 负标志 | 结果为负（最高位 = 1）|
| **V (Overflow)** | 溢出标志 | 有符号数运算超出范围 |

## 2.7 控制单元（CU）的工作

### 硬连线控制 vs 微程序控制

| | 硬连线 | 微程序 |
|---|--------|--------|
| **实现方式** | 固定逻辑电路（门电路） | 微指令存储在 ROM 中 |
| **速度** | 快（直接电路） | 慢（多读一级 ROM） |
| **灵活性** | 低（改指令 = 改硬件） | 高（改微程序即可） |
| **成本/复杂度** | 高 | 低 |
| **典型应用** | RISC 处理器（ARM） | CISC 处理器（x86） |

> 现代 CPU 往往混合使用：RISC 风格的内核 + 微码处理复杂/不常用指令。
