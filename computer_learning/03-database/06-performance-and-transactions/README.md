### ACID 速记

```
A - 原子性：要么全做，要么全不做
C - 一致性：数据永远符合约束
I - 隔离性：并发事务感觉不到彼此
D - 持久性：提交了就不会丢
```

### 索引使用原则

```
✅ 适合建索引：
  - WHERE / JOIN / ORDER BY 频繁使用的列
  - 选择性高的列（值分散）
  
❌ 不适合建索引：
  - 频繁更新的列
  - 选择性低的列（如性别）
  - 小表（全表扫描更快）
```
